<h2><strong>Задание: поиск потомка</strong></h2>

<p>Откройте страницу <a href="http://suninjuly.github.io/cats.html" rel="nofollow noopener noreferrer">http://suninjuly.github.io/cats.html</a>. Откройте консоль разработчика&nbsp;и вкладку Elements в ней. Напишите <strong>минимально</strong> достаточный CSS-селектор, который найдет элемент с картинкой серьезного кота (Serious cat). Для поиска подходящего элемента в каталоге картинок используйте родительский элемент <strong>div.col-sm-4 </strong>вместе с&nbsp;псевдо-классом<strong>&nbsp;:nth-child(n)</strong>, чтобы выбрать n-й по счету элемент, а также&nbsp;селектор для картинки по тегу <strong>img</strong>. При написании этого селектора не надо использовать символ <strong>&gt;</strong>, т.к. это задача на поиск&nbsp;потомка&nbsp;без использования дочерних элементов.&nbsp;Для решения достаточно тех селекторов, которые уже указаны в задании.</p>

<p>Пожалуйста, не используйте селекторы, которые генерирует браузер по кнопке &quot;скопировать css селектор&quot; или расширения. Зачастую они строят полный путь, начиная от <strong>body</strong>&nbsp;&mdash; а это очень нестабильный селектор, писать такие в своем коде это плохая практика. При малейшем изменении структуры страницы все ваши селекторы потеряют актуальность.&nbsp;</p>

<p>Не используйте в синтаксисе квадратные скобки&nbsp;&mdash; конструкции вида&nbsp;<em>[class=&quot;stepic&quot;].</em></p>

<p>Введите найденный селектор в качестве ответа на это задание.</p>
