<h2>Test structure</h2>

<p>When writing UI tests, you can use the same Python features as in the case of unit tests created by developers. </p>

<p>Any test should contain:</p>

<ol>
	<li>Input data.</li>
	<li>A test scenario, i.e., a sequence of steps to be executed to get the result.</li>
	<li>Result check.</li>
</ol>

<p>Let's discuss how we can do the checks. </p>
